# 1. "오늘" 판정 — author date + 반열림 로컬 구간

- Status: Accepted
- Date: 2026-06-08
- Source: PRD (#1), Q2 · Q3

## Context

"오늘 얼마나 썼나"를 세려면 "오늘"과 "어느 시각"을 정의해야 한다. 타임존, rebase/amend로
끌려온 과거 작업, 클럭 스큐(미래 타임스탬프), DST 전환일 같은 함정이 숫자를 왜곡할 수 있다.

## Decision

- **날짜 기준은 author date.** committer date가 아니다.
- "오늘" = 머신 **로컬 타임존**의 반열림 구간 `[오늘 자정, now)` (하한 닫힘, 상한 열림).
- 멤버십은 각 커밋의 **UTC 인스턴트**를 이 구간과 비교해 판정한다. 커밋에 기록된 타임존
  offset은 멤버십에 무관하다 — 비교는 절대시각으로만.
- 이 로직은 순수 함수 `Window`로 격리한다(IO 없음).

## Consequences

- 어제 짠 걸 오늘 rebase/amend해도 어제 작업이 오늘로 부풀지 않는다.
- 미래 author 타임스탬프(클럭 스큐)는 `now` 밖이라 자동 제외.
- 자정 정각 커밋은 포함, 현재 시각(now) 커밋은 제외.
- DST 전환일의 하루 길이 변동(23/25시간)은 그대로 수용한다.
- 가장 까다로운 타임존/자정/반열림 로직이 `Window` 한 곳에 모여 결정적으로 테스트된다.
